Erreur lors de la publication de sites web asp.net 5 (core) vers Azure

Feb 3, 2016 1 min.

Si vous faites du développement asp.net core et que vous essayez de publier votre site web vers un azure website il se peut que vous ayez rencontré l’erreur suivante en suivant cette documentation https://msdn.microsoft.com/en-us/Library/vs/alm/Build/azure/deploy-aspnet5?f=255&MSPPError=-2147217396 [error]Error: The specified credentials cannot be used with the authentication scheme ‘Basic’. [error]Error: Default credentials cannot be supplied for the Basic authentication scheme. [error]Parameter name: authType Cela vient du fait que le script fournit dans la documentation ne supporte pas les sites qui ont des slots de déploiement.

Migration Git – nettoyage des références TFS

Jan 11, 2016 1 min.

J’ai récemment dû migrer plusieurs contrôles de code source depuis TFS vers Git (Git sur vsonline). Pour ce faire j’ai utilisé Git-Tfs https://github.com/git-tfs/git-tfs qui fait très bien le travail et est vraiment bien documenté https://github.com/git-tfs/git-tfs/blob/master/doc/usecases/migrate_tfs_to_git.md A une étape de la documentation on vous conseille de supprimer les fichiers *.vspscc, *.vssscc et de nettoyer les fichiers *.sln et *.*proj des références vers TFS. Ceci permet d’éviter que Visual Studio tente de se reconnecter à TFS à chaque fois que l’on ouvre projet/solution.
